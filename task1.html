What is the difference between a normal div and a grid container?
Div is Just a simple block element. No layout system by default. You manually control position using margin, padding, float, etc. where as Grid A div that becomes a layout manager. It arranges child elements into rows and columns automatically.
Why does the layout look the same even after applying Grid?
Grid is applied, but no grid structure is defined, so layout stays same.
Are grid items created automatically?
yes grid items are created automatically. When you apply display: grid to a container, all its direct child elements automatically become grid items without needing any extra class or property. You only control their position using grid rules, not their creation.
Can Grid work without rows or columns?
Yes, Grid works without defining rows or columns, but the browser auto-creates a simple single-column layout, so it looks normal.
What do you think Grid needs to show visible changes?
Grid needs defined properties like rows and columns or the gap between them to visbily look like a grid.
